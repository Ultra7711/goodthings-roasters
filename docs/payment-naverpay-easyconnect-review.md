# 네이버페이 · 이지커넥트 도입 검토 보고서

> 작성일: 2026-06-18 · 대상: Good Things Roasters 온라인몰 + 오프라인 카페
> 상태: **검토(의사결정 전)** · 비용·수수료는 **공식 상담으로 확정 필요**(본문 §7·§10)

---

## 1. 목적

토스페이먼츠 라이브 심사가 진행 중인 상황에서, **네이버페이/이지커넥트를 (토스보다) 먼저 붙여 오픈을 앞당기는 것**, 그리고 **오프라인 카페를 함께 운영하는 굳띵즈에게 네이버 생태계 시너지가 실재하는가**를 판단하기 위한 검토.

## 2. 용어 정리 (혼동 주의 — 셋은 서로 다른 것)

| 명칭 | 정체 | 굳띵즈 관련 |
|------|------|------------|
| **이지커넥트** (easyconnect.naver.com) | 자체몰에 네이버 비즈니스 도구(네이버페이·로그인·톡톡·광고)를 묶어 연동시켜 주는 **유치 프로그램** (연동·광고비·컨설팅 무료 지원) | **온라인몰**(이 프로젝트)에 직접 적용 |
| **네이버페이** (주문형/결제형/종합형) | 자체몰 **온라인 결제** 가맹 | 온라인몰 결제수단 |
| **네이버페이 커넥트(CONNECT)** | 2025.6 출시 **오프라인 매장 결제 단말기** | **오프라인 카페** POS |

→ 온·오프 시너지를 완성하려면 **이지커넥트(온라인) + 커넥트 단말(오프라인) + 네이버 플레이스**가 함께 묶여야 함. 본 보고서는 온라인몰(이지커넥트/네이버페이) 중심, 오프라인 연계는 §6.

## 3. 현황 (검토 출발점)

- **온라인몰**: 자체 Next.js 쇼핑몰. **결제는 토스페이먼츠로 깊이 통합 완료**(`paymentService`·confirm RPC·webhook·3중 멱등·정기배송 billing). 코드는 토스 전용.
  - 단, 코드의 간편결제 매핑 테이블에 **네이버페이(`naverpay`)가 이미 등록**돼 있음 → **토스페이먼츠를 통하면 네이버페이가 간편결제 수단으로 자동 포함**(카카오페이·삼성페이 등도). 단 이는 토스 심사 통과가 전제.
- **결제 라이브**: 토스 **심사 중**(통상 수주~). 미통과 = 오픈 불가.
- **오프라인**: 카페 매장 운영 중 + 네이버 플레이스 등록(S320에서 지도/플레이스 링크 작업).
- **호스팅**: 별도 검토 중(Hobby 상업 불가 → Pro/셀프호스팅 결정 예정).

## 4. 옵션 정의

| 옵션 | 내용 | 오픈 시점 |
|------|------|-----------|
| **A. 토스 유지·대기** | 현 토스 통합 그대로, 심사 완료까지 대기. 네이버페이는 토스 간편결제로 자동 포함 | 토스 심사 종료 시 |
| **B. 네이버페이 먼저 가맹** | 네이버페이 직접 가맹(심사 2~3영업일~1주)으로 먼저 오픈. 토스는 이후 추가 | 빠름(단 개발 공수) |
| **C. 종합형 + 토스 병행** | 네이버페이 종합형 + 토스를 모두 운영 | 가장 늦음/복잡 |

## 5. 분석 축

### 5.1 오픈 속도
- 네이버페이 직접 가맹 심사: **결제형 2~3영업일, 주문형 최대 1주일**.
- 토스: **심사 중**(통상 더 김·반려 리스크 존재).
- ⚠️ **단 현재 결제 플로우는 토스 전용**(confirm/webhook/멱등/정기배송). 네이버페이 직접 가맹(옵션 B)은 **결제 연동 재구현 공수**가 발생 → "심사 속도 이득"을 개발 기간이 일부 상쇄.
- **결정 변수**: `토스 잔여 심사기간` vs `네이버페이 재구현 공수`. 토스가 곧 끝나면 B는 손해, 길어지거나 반려되면 B가 탈출구.

### 5.2 비용 (⚠️ 전부 공식 확인 필요 — §10)

| 항목 | 토스페이먼츠 | 네이버페이 (이지커넥트) |
|------|-------------|------------------------|
| 가입비 | 22만 원(최초 1회)으로 알려짐 | 이지커넥트 **연동 무료 지원** 표방 |
| 연(관리)비 | 11만 원/년으로 알려짐 (초기 합계 33만·VAT 포함) | 광고비·컨설팅 무료 지원 표방 |
| 호스팅사 경유 | **가입비·연회비 면제 가능**(경로별) | — |
| 결제 수수료 | 카드 약 3.4%, 영세·중소 우대 별도(VAT 별도) | 결제형 평균 약 2.3%, 영세 약 1.98%(자료별 상이) |

- **고정비는 네이버페이가 유리**(이지커넥트 무료 지원). 단 "무료"의 **기간·한도·조건**은 미확정.
- **수수료는 등급·유형·경로별로 달라** 단정 불가. 양쪽 **실요율을 같은 조건으로 견적** 받아야 비교 가능.
- 네이버페이 **포인트 적립**은 가맹점 일부 부담 구조일 수 있음 → "수수료 너머 실비용" 확인 필수.

### 5.3 자체몰 운영권 (주문형 vs 결제형 — 시너지의 핵심 트레이드오프)

| | 주문형 | 결제형 | 종합형 |
|---|--------|--------|--------|
| 결제 위치 | **네이버 주문서**(네이버 ID로 가입 없이) | **자사몰 주문서** 내 결제수단 | 둘 다 |
| 대상 상품 | 실물배송만 | 실물+비실물 | 둘 다 |
| 운영 주도 | **네이버 정책**(주문·배송·CS·리뷰·적립) | **자체몰 정책** | 혼합 |
| 네이버 시너지 | **강**(검색·노출·회원·리뷰·적립) | 약(결제수단 추가) | 강 |
| 자체몰 가치 | **약화**(고객이 네이버 주문서로 이탈) | **유지** | 절충 |

→ 굳띵즈는 **자체 Next.js 몰을 공들여 구축**했음(리뷰·정기배송·디자인 시스템). **주문형은 그 자산을 우회**시켜 운영권·데이터·UX 통제를 네이버에 넘김. **결제형이 자체몰 가치를 지키는 선택**이나 시너지는 약함. **종합형**이 절충이지만 복잡도·이중관리 부담.

### 5.4 개발 공수
- 옵션 A(토스 대기): **0** (이미 완성).
- 옵션 B(네이버페이 결제형 직접): confirm/webhook/멱등/정기배송에 대응하는 **네이버페이 연동 신규 구현**. 정기배송(구독) 네이버페이 지원 여부가 핵심 변수(굳띵즈 핵심 기능).
- 옵션 C(종합형+토스): 가장 큼.

### 5.5 락인·리스크
- 네이버 방향 = 검색·페이·멤버십·플레이스 **생태계 종속** 심화. 이탈 비용↑.
- 주문형은 **고객 데이터·관계가 네이버에 귀속**되는 측면.
- 신규(커넥트 단말 2025.6) 레퍼런스 짧음.

## 6. 오프라인 시너지 (운영 중인 카페 — 사용자 핵심 관심)

네이버의 강점은 **온·오프 통합 고객 여정**이며, 토스(결제 처리 단일)에는 없는 영역:

| 접점 | 네이버 생태계 | 토스 |
|------|--------------|------|
| 매장 발견 | 네이버 검색 → **플레이스** → 방문 | ❌ |
| 온라인 결제 | 이지커넥트(네이버페이·로그인·톡톡·광고) | 결제만 |
| 오프라인 결제 | **네이버페이 커넥트 단말**(기존 POS 연동) | 토스 단말 |
| 통합 적립·멤버십 | **온·오프 동일 네이버 ID로 적립·리뷰·재방문** | ❌ |
| 데이터 | 검색→방문→구매→리뷰 **단일 여정** | 결제 단계만 |

→ **온·오프를 모두 운영하는 굳띵즈에겐 시너지가 실재**. 단 완성하려면 온라인(이지커넥트)+오프라인(커넥트 단말)+플레이스를 함께 묶어야 하고, **락인도 그만큼 커짐**. 또한 **네이버페이 적립의 가맹점 부담**이 온·오프 양쪽에 누적될 수 있어 비용 확인이 전제.

## 7. 의사결정 시나리오

| 우선 목적 | 권고 |
|-----------|------|
| **빠른 오픈만** | 토스 잔여 심사기간 먼저 확인. 곧 종료면 **A 유지**(재구현 손해). 길거나 반려 위험이면 **B(네이버페이 결제형) 직접 가맹**으로 우회 |
| **온·오프 시너지** | 네이버 방향. 단 **결제형**(자체몰 가치 보존)을 기본으로, 오프라인 커넥트 단말 + 플레이스 연계. 주문형은 자체몰 자산 훼손이라 **비권장** |
| **안정·통제** | **A(토스)** + 토스 경유 네이버페이 간편결제(이미 코드 매핑 존재). 자체몰 운영권 100% 유지 |

## 8. 권고 (요약)

1. **자체몰 가치를 지키는 게 우선** → 네이버페이를 도입하더라도 **결제형**(주문형 ✕). 굳띵즈의 리뷰·정기배송·디자인 자산을 네이버 주문서로 우회시키지 말 것.
2. **"빨리"가 진짜 목적이면** 토스 잔여 심사기간을 먼저 확인. 재구현 공수 vs 대기시간 비교가 핵심.
3. **시너지가 진짜 목적이면** 온·오프 통합은 네이버가 우위 — 단 **적립 부담·수수료·락인**을 무료 컨설팅으로 정량 확인 후.
4. **지금 당장 할 일은 정보 수집(공수 0)**: 이지커넥트 무료 컨설팅으로 §10 항목 확정. 실연동은 호스팅·토스 라이브 결정과 함께.

- 토스 잔여 심사기간·반려 가능성
- 네이버페이 직접 가맹 시 결제 재구현 실제 공수 산정

- [ ] **포인트·적립 가맹점 부담** 구조 및 비율
