# GTR QA 시나리오 매트릭스

> 실제 발견된 버그 패턴 기반 수동 테스트 체크리스트.
> 새 버그 수정 후 관련 시나리오를 추가한다. 버그 번호는 `docs/bug-and-polishing.md` 참조.
>
> **기기 표기:** D = Desktop · M = Mobile(iOS/Android) · i = iOS 전용 · a = Android 전용

---

## 주의 패턴 (빠른 참조)

이 프로젝트에서 반복 발생한 버그 원인 7가지. 새 코드 작성 후 해당 패턴 범주를 먼저 확인한다.

| 패턴 | 위험 영역 | 대표 버그 |
|------|----------|----------|
| **cacheComponents 상태 잔류** | 페이지 이탈 후 재진입 시 이전 상태가 남아있음 | BUG-112, 오늘 수정 |
| **인증 상태 전환** | 로그인/복귀/자동로그인/세션 isLoading 구간 | BUG-112, BUG-156 |
| **이벤트 관통** | 드롭다운·드로어·오버레이 외부 탭 | BUG-125, BUG-154, BUG-158 |
| **autofill** | Chrome/Safari 자동완성 — 배경색·필드값·라인 소실 | BUG-106, BUG-114, BUG-151, BUG-152 |
| **모바일 레이아웃** | safe-area·svh·iOS flex quirk·브라우저 네비바 | BUG-103~105, BUG-148 |
| **뒤로가기** | 브라우저 back 버튼·앱 내 back | BUG-133, BUG-149 |
| **드로어 ↔ 페이지 전환** | 드로어 닫힘 타이밍·잔상·지연 | BUG-149, BUG-150 |

## 1. 결제 플로우

### 1-1. 진입 조건별 1단계 UI

| 조건 | 기대 결과 | 버그 참조 |
|------|----------|----------|
| 로그인 상태 → 장바구니 → 주문하기 | 로그인/비회원 선택 없이 폼 바로 표시 | BUG-112 |
| 비로그인 → 장바구니 → 주문하기 | 로그인 / 비회원으로 주문하기 선택 표시 | — |
| 비로그인 → `/checkout` URL 직접 진입 (카트 있음) | 로그인 / 비회원 선택 표시 | — |
| 비로그인 → `/checkout` URL 직접 진입 (카트 비어있음) | 빈 카트 UI (아이콘 + 문구 + 쇼핑 계속하기 버튼) | BUG-155 |

### 1-2. cacheComponents 재진입 (중요)

- [ ] 비회원으로 주문하기 클릭 (폼 열림) → 드로어 닫음 → 다시 주문하기 → **로그인/비회원 선택 화면** 표시 (폼 리셋 확인) `D·M`
- [ ] 위와 동일 시나리오를 이메일까지 입력 후 이탈해도 동일 결과 `M`
- [ ] 비회원 폼 열기 → 다른 페이지(/shop 등) 이동 → 장바구니 → 주문하기 → 폼 리셋 확인 `D·M`

### 1-3. 비회원 → 로그인 → 복귀

- [ ] 비회원 결제 1단계 → "로그인하고 주문하기" → 로그인 성공 → `/checkout` 복귀 → **비회원 UI 없음**, 폼 바로 표시 `D·M` (BUG-112)
- [ ] 위 경로에서 로그인 페이지에 "비회원 주문" 박스 없음 확인 `D·M`

### 1-4. autofill

- [ ] 결제 폼 → 브라우저 자동완성 수락 → 인풋 배경 **파란색 없음** `M` (BUG-106)
- [ ] 자동완성 수락 후 인풋 하단 **구분선 표시됨** `M` (BUG-152)
- [ ] 우편번호 필드 — 자동완성으로 값 주입 안 됨 (Daum Postcode 검색으로만 입력) `D·M` (BUG-114)
- [ ] 비밀번호·비밀번호 확인 필드 — 진입 시 미리 채워진 값 없음 `D·M` (BUG-151)

### 1-5. 폼 검증

- [ ] 비회원 비밀번호 헬퍼 텍스트 자리수와 에러 메시지 자리수 동일 `D·M` (BUG-113)

## 2. 인증 / 세션

### 2-1. 헤더 상태 플리커

- [ ] 브라우저 종료 후 재방문 → 헤더 로그인 아이콘이 **세션 확정 전 깜빡이지 않음** (아이콘 숨김 유지 후 확정) `D·M` (BUG-156)
- [ ] 재방문 시 자동 로그인되면 헤더가 즉시 로그인 상태 반영 `D·M`

### 2-2. 로그인 후 복귀

- [ ] 로그인 성공 → 메인 복귀 → 어나운스 바 정상 위치 (스크롤 offset 없음) `D·M` (BUG-136)
- [ ] 로그인 성공 → `/checkout` 복귀 → 비회원 UI 없음 `D·M`

### 2-3. 로그아웃

- [ ] 로그아웃 → `/checkout` 진입 → 비회원/로그인 선택 표시 (이전 세션 폼 값 없음) `D·M`

## 3. 장바구니 드로어

### 3-1. 열기 / 닫기

- [ ] 드로어 열린 상태에서 브라우저 back → **드로어만 닫힘** (페이지 이동 없음) `M` (BUG-133)
- [ ] 드로어 닫기 후 다시 열기 → 이전 스크롤 위치 초기화 `D·M`

### 3-2. 드로어 → 풀페이지 전환

- [ ] 드로어 → "장바구니 보기" → 전환 중 **상품 상세 페이지 콘텐츠 노출 없음** `D·M` (BUG-149)
- [ ] 위 전환 속도 — 체감상 빠름 (350ms 내외) `D·M` (BUG-150)

### 3-3. 결제 진입 후 재진입

- [ ] 드로어 → 주문하기 → 이탈 → 드로어 → 주문하기 → 체크아웃 1단계 **리셋 상태** 확인 `D·M`

### 3-4. 모바일 레이아웃

- [ ] 모바일 드로어 CTA 버튼 두 개 **1:1 비율** `M` (BUG-122)
- [ ] 드로어 본체와 하단 푸터 **좌우 패딩 일치** `M` (BUG-123)
- [ ] 하단 CTA 영역이 브라우저 네비바에 **가려지지 않음** `M` (BUG-121)

## 4. 모바일 특화

### 4-1. 레이아웃

- [ ] 상품 상세 페이지 — **상단부터** 렌더링 (푸터 먼저 노출 없음) `i` (BUG-148)
- [ ] 히어로 하단이 모바일 브라우저 네비바에 **가려지지 않음** `M` (BUG-105)
- [ ] 메뉴·쇼핑·굿데이즈 페이지 — 타이틀 영역이 헤더 아래 적정 위치 `M` (BUG-134)

### 4-2. iOS 특화

- [ ] 인풋 필드 **하단 구분선 표시됨** (고DPR hairline) `i` (BUG-103)
- [ ] 카페 메뉴 바텀시트 하단 — safe-area 배경 흰색 노출 없음 `i` (BUG-104)
- [ ] 저전력 모드 — 히어로 비디오 **첫 스크롤(touchstart)에서 재생** `i` (BUG-160)
- [ ] 모달/드로어/검색 패널 열린 상태에서 배경 **scroll 관통 없음** `i` (BUG-138)

### 4-3. 터치 인터랙션

- [ ] 버튼 위로 손가락이 지나갈 때 (드래그/스크롤) 호버 애니메이션 **발화 없음** `M` (BUG-144)
- [ ] 버튼 탭 → 골드 라인 연출 → 액션 실행 순서 `M` (BUG-143)

## 5. 이벤트 관통

> 공통 검증 원칙: 오버레이·드롭다운·패널 외부 탭은 **해당 레이어만 닫히고** 하단 엘리먼트 액션은 소비되어야 한다.

- [ ] 마이페이지 드롭다운 열기 → 외부 탭 → 드롭다운만 닫힘 (하단 아코디언 동작 없음) `D·M` (BUG-125)
- [ ] 상품 카드 빠른 추가 활성화 → 배경 탭 → 빠른 추가만 닫힘 (다른 상품 카드 클릭 없음) `D·M` (BUG-154)
- [ ] 비즈니스 폼 드롭다운 열기 → 외부 버튼 탭 → 드롭다운만 닫힘 `D·M` (BUG-158)
- [ ] 검색 패널 열기 → 외부 탭 → 패널만 닫힘 + 배경 스크롤 없음 `M` (BUG-138)

## 6. 네비게이션 / 헤더

- [ ] 다크 테마 페이지 ↔ 라이트 테마 페이지 전환 — 헤더만 먼저 색이 바뀌는 **깜빡임 없음** `D·M` (BUG-130)
- [ ] 페이지 간 네비게이션 후 어나운스 바 **정상 위치** (scrollTop offset 없음) `D·M` (BUG-136)

## 7. 회귀 위험 높은 항목

> 수정 이력이 2회 이상인 항목. 관련 코드 수정 후 반드시 재검증.

| 항목 | 확인 포인트 | 버그 |
|------|------------|------|
| `chp-input` 배경 | `background: #FBF8F3` 유지 확인 (`transparent` 금지) | BUG-106 |
| 비회원 UI 조건 | `!isFormRevealed && !isLoggedIn && !sessionLoading` 3중 조건 | BUG-112 |
| 헤더 테마 전환 | 새 라우트 추가 시 `headerThemeConfig` 등록 여부 | BUG-130 |
| `data-gtr-tap` | 새 버튼 추가 시 탭 피드백 어트리뷰트 누락 여부 | BUG-143 |

## 신규 시나리오 추가 가이드

버그 수정 후 관련 재현 경로를 아래 형식으로 추가한다.

```
- [ ] {진입 경로} → {기대 결과} `{기기}` (BUG-XXX)
```

- 기기: `D` = Desktop, `M` = Mobile, `i` = iOS only, `a` = Android only
- 기대 결과는 "무엇이 일어나야 하는가" 로 작성 (무엇이 일어나지 않아야 하는지 포함)
- 관련 버그 번호 필수 기재 (맥락 추적용)
