# 토스페이먼츠 빌링(자동결제) 통합 — 클라이언트 안내

**작성:** 2026-05-07
**목적:** 정기배송 자동결제 도입에 앞서, 클라이언트 컨펌이 필요한 4가지 비즈니스 결정과 토스 비즈센터 심사 신청 안내
**참조:** docs/adr/ADR-008-toss-billing-integration.md (기술 마스터 플랜)

---

## 1. 왜 필요한가

현재 정기배송은 **첫 회차 결제만 작동**합니다. 다음 배송일이 도래해도 자동결제·발송 로직이 없어 사실상 "1회성 결제 + 정기배송 라벨" 상태입니다.

**토스 빌링 도입 후:**
- 사용자가 카드(또는 계좌)를 **한 번 등록** 하면 매 사이클마다 **자동 결제 + 자동 발송**
- 카드 만료 30일 전 자동 알림, 결제 실패 시 재시도·일시중지 자동 처리
- 사용자는 마이페이지에서 카드 변경·해지 가능

## 2. 토스 비즈센터 심사

토스 빌링은 **별도 계약·심사**가 필요합니다.

- 굳띵즈 카페에서 이미 **토스 키오스크 결제를 사용 중**이므로 심사 자체는 어렵지 않을 것으로 예상됩니다.
- 심사 신청 → 통과 → 라이브 키 적용 까지 약 2~3주 소요(예상).
- 심사 신청은 **클라이언트 컨펌 후** 진행합니다.

**개발팀 작업 병행 가능:** 코드 통합은 sandbox 키로 미리 진행하여, 심사 통과 시점에 즉시 운영 전환할 수 있도록 준비합니다.

## 3. 컨펌 받아야 할 결정 4건

### D-2. 정기배송 결제수단

**제안:** 카드 + 계좌이체(퀵계좌이체)

- 토스 빌링은 **간편결제(토스페이·카카오페이) 미지원** — 정기배송에서는 제외
- 일반 1회성 결제는 간편결제 그대로 지원

→ **이대로 진행해도 될까요?**

### D-3. 정기 + 일반 상품이 한 카트에 있을 때

**제안:** 사용자는 카드를 **두 단계로 등록**

1. 정기배송용 카드 등록(빌링키 발급) → 첫 회차 자동 결제
2. 일반 상품용 위젯 결제(단발)

이유: 사용자가 카드를 한 번에 입력하면 토스 정책상 빌링과 일반결제를 분리할 수 없습니다. 두 단계로 나눠야 카드 정보가 빌링 전용으로 안전하게 저장됩니다.

→ **이대로 진행해도 될까요?** 만약 "한 번에 결제" 우선이라면 첫 회차도 위젯 결제로 처리하고 다음 회차부터 빌링 적용하는 대안 가능(단, 사용자가 카드를 두 번 입력해야 함).

### D-4. 베타 사용자 데이터 처리

**제안:** 운영 시작 시 **현재 DB의 모든 정기배송·주문 데이터 초기화**

- 현재 데이터는 모두 개발 테스트 데이터입니다.
- 운영 시작 시점에 clean slate로 출발하는 것이 데이터 정합성·고객 경험 모두 안전합니다.

→ **혹시 베타 사용자 데이터 보존이 필요하다면 알려주세요.**

### D-5. 정기배송 가입 자격

**제안:** **회원만 정기배송 가능**

- 게스트(비회원)는 정기배송 카트에 담을 때 회원가입 안내 모달 표시
- 일반 1회성 결제는 게스트 그대로 가능

이유: 빌링키는 사용자 ID(customerKey)와 1:1 매칭되어야 하며, 자동결제·카드 관리·해지 흐름 모두 회원 계정 기반에서만 안전하게 동작합니다.

→ **이대로 진행해도 될까요?**

## 4. 일정 영향

| 단계 | 소요 | 비고 |
|------|------|------|
| 클라이언트 컨펌 (위 4건) | 1주 이내 | 빠를수록 좋음 |
| 토스 비즈센터 심사 신청·통과 | 2~3주 | 클라이언트 명의로 신청, 개발팀 지원 |
| 개발 (sandbox) | 5~7 sprint | 컨펌 받은 즉시 시작, 심사와 병행 |
| 운영 전환 (라이브 키 + 데이터 초기화) | 0.5 sprint | 심사 통과 + 개발 완료 시점 |

전체적으로 **컨펌 → 5~7주** 후 정기배송 정식 출시 가능 예상.

## 5. 클라이언트 회신 요청 항목

다음 5가지에 대한 회신 부탁드립니다:

1. **D-2 결제수단** — 카드 + 계좌이체로 진행해도 되는지
2. **D-3 혼합 카트** — 빌링 + 위젯 분리 두 단계 입력 OK 인지 (또는 대안 선호)
3. **D-4 데이터 초기화** — 운영 전환 시 베타 데이터 truncate OK 인지
4. **D-5 회원 강제** — 게스트 정기배송 차단 OK 인지
5. **토스 비즈센터 심사 신청 시점** — 위 4건 컨펌 받는 즉시 신청 진행해도 되는지 (계정 명의·서류 준비 협조 필요)

회신 받는 즉시 개발 착수하고, 심사 진행 상황은 별도 공유 드립니다.
