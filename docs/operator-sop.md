# 운영자 SOP — Good Things Roasters 어드민 운영 매뉴얼

> **대상:** 사이트를 일상적으로 운영하는 사업자·운영 담당자 (비개발자)
> **범위:** 어드민(`/admin`)에서 수행하는 모든 운영 행위의 표준 절차
> **갱신 기준:** 어드민 기능 정의(SoT)는 `docs/admin-implementation-plan.md`. 본 문서는 "운영자가 무엇을 어떻게 클릭하는가"의 행위 절차 관점이며, 기능 명세와 별개로 유지한다.

---

## 0. 시작하기 전에

### 0-1. 로그인

1. 브라우저에서 **`/admin/login`** 접속 (일반 회원 로그인과 별도 화면).
2. 관리자 권한이 부여된 계정의 이메일·비밀번호로 로그인.
3. 권한이 없는 계정으로 접근하면 로그인 화면으로 되돌려 보낸다.

### 0-2. 권한 단계 (중요)

운영 계정은 두 단계로 나뉜다. 화면에서 가능한 작업이 다르다.

| 단계 | 한글 표기 | 할 수 있는 일 |
|------|----------|--------------|
| **owner** | 관리자 | **모든 작업** + 회원 권한 관리 · 회원/구독자 명단 내보내기 · 뉴스레터 캠페인 발송 · 계정 강제 삭제 · 감사 로그 |
| **staff** | 운영자 | 주문·정기배송·문의·상품·메뉴·배너·설정 등 **일상 운영 전반** (위 owner 전용 항목 제외) |

> 본 문서에서 🔒 표시는 **owner(관리자)만** 가능한 작업이다. staff(운영자)는 해당 버튼이 막혀 있거나 보이지 않는다.

### 0-3. 화면 구조

- **왼쪽 사이드바** — 4개 그룹으로 메뉴가 묶여 있다.
  - **운영**: 대시보드 · 주문 · 정기배송
  - **카탈로그**: 상품 · 카페 메뉴 · 고객 · 굿데이즈
  - **인사이트**: 통계 · 뉴스레터 · 비즈 문의
  - **설정**: 사이트 설정 · 배너 · (감사 로그 🔒)
- **상단 바** — 현재 페이지 제목 + **🔔 알림 벨** + 계정 아바타.

### 0-4. 알림 벨 — 매일 가장 먼저 확인

상단 🔔 아이콘에 빨간 숫자(배지)가 떠 있으면 처리할 일이 있다는 뜻이다. 클릭하면 **"할 일"** 목록이 열리고, 항목을 누르면 해당 화면으로 바로 이동한다.

| 알림 항목 | 의미 | 이동 위치 |
|-----------|------|-----------|
| **발송 대기 주문** | 결제 완료됐으나 아직 발송 처리 안 한 주문 | 주문 > 신규 탭 |
| **환불 요청** | 고객이 환불을 신청한 주문 | 주문 > 환불신청 탭 |
| **송장 누락** | 발송 처리됐는데 송장번호가 비어 있는 주문 | 주문 > 송장누락 탭 |
| **신규 비즈문의** | 아직 응대 안 한 사업 제휴/대량주문 문의 | 비즈 문의 > 신규 |

> 알림은 페이지를 이동하거나 창으로 돌아올 때 자동 갱신된다. 배지 숫자가 0이면 급한 처리 항목이 없다.

## 1. 주문 관리

사이드바 **운영 > 주문**.

### 1-1. 주문 상태 이해하기

주문은 다음 흐름을 따른다.

```
결제대기 → 신규(결제완료) → 배송중 → 완료
                  └→ (고객 환불 신청 시) 환불신청 → 환불처리 → 환불완료
```

| 상태 | 뜻 | 운영자 할 일 |
|------|-----|-------------|
| **결제대기** | 결제가 아직 안 끝남 (가상계좌 입금 대기 등) | 입금 완료를 기다린다 |
| **신규** | 결제 완료, 발송 전 | **발송 처리** 필요 (→ 1-3) |
| **배송중** | 송장 등록 완료 | 배송 진행 (자동) |
| **완료** | 배송 완료 | — |
| **취소 / 환불신청 / 환불처리 / 환불완료** | 취소·환불 진행 단계 | 환불은 Toss 콘솔에서 (→ 2장) |

### 1-2. 주문 찾기 (검색·필터)

주문 목록 상단에서:

- **검색창**: 주문번호, 고객명, 이메일로 검색.
- **상태 탭**: 전체 / 신규 / 배송중 / 완료 / 취소 / 환불신청 / 송장누락.
- **기간 필터**: 전체 기간 / 최근 7일 / 30일 / 90일.
- **결제수단 필터**: 전체 / 카드 / 계좌이체.

### 1-3. 발송 처리 (가장 중요한 일상 작업)

1. **주문** 목록에서 **신규** 탭을 연다 (또는 알림 벨의 "발송 대기 주문").
2. 발송할 주문을 클릭해 **주문 상세**로 들어간다.
3. 우측 상단 **[발송 처리]** 버튼을 누른다. (결제 완료 주문에서만 활성화됨)
4. **발송 처리** 다이얼로그에서:
   - **택배사**: 드롭다운에서 선택 (CJ대한통운 · 한진택배 · 롯데택배 · 로젠택배 · 우체국택배). 목록에 없으면 **직접입력...**을 골라 택배사 이름을 입력.
   - **송장번호**: 대시(-) 없이 숫자만 입력. (12자 이상 권장)
5. **[발송 처리]** 버튼을 누른다.

> ✅ 발송 처리하면 주문 상태가 **배송중**으로 바뀌고, **고객에게 배송 알림 메일이 자동 발송**된다. 송장번호는 고객의 마이페이지 주문내역에도 표시된다.
>
> ⚠️ 이미 발송 처리한 주문은 다시 처리할 수 없다("이미 발송 처리된 주문입니다" 안내가 뜬다). 송장을 잘못 넣었다면 아래 송장 누락 항목과 동일하게 개발 담당에게 문의.

### 1-4. 관리자 메모

주문 상세 우측 **메모** 영역에 내부용 메모를 적을 수 있다 (고객에게는 보이지 않음). 입력 후 저장한다.

### 1-5. 주문 내역 내보내기

주문 목록 화면에서 **엑셀 내보내기** 기능으로 현재 조건의 주문 목록을 `.xlsx` 파일로 받을 수 있다 (정산·집계용).

## 2. 환불 · 취소 처리 ⚠️ (어드민 외부 — Toss 콘솔)

> **핵심: 환불은 어드민에서 직접 하지 않는다. Toss(토스페이먼츠) 콘솔에서 처리한다.**

어드민의 주문 상세에는 **환불 · 취소** 안내 섹션만 있고, 실제 환불 버튼은 없다. 이는 결제 취소가 토스페이먼츠를 통해 이뤄지기 때문이다.

### 절차

1. **Toss 콘솔(토스페이먼츠 상점관리자)**에 로그인한다.
2. 해당 결제 건을 찾는다 (주문 상세에 표시된 주문번호·결제 정보로 대조).
3. **전체 환불 또는 부분 환불**을 선택하고 사유를 입력한다.
   - 가상계좌(계좌이체) 결제는 **고객의 환불 받을 계좌 정보**가 추가로 필요하다.
4. Toss에서 환불을 실행하면, 잠시 후 어드민 주문 상태가 **취소 / 환불처리 / 환불완료**로 자동 반영된다.

> 참고 가이드: Toss "결제 취소하기" 문서 — `https://docs.tosspayments.com/guides/v2/cancel-payment`
>
> 어드민 주문 상세의 **환불 · 취소** 섹션에도 동일한 Toss 가이드 링크가 있다.

## 3. 정기배송 관리

사이드바 **운영 > 정기배송**.

### 상태 이해

| 상태 | 뜻 |
|------|-----|
| **진행중** | 정상 구독 중 |
| **일시정지** | 고객/운영자가 일시 중단 |
| **해지** | 구독 종료 |
| **만료** | 결제 실패 등으로 자동 만료 |

### 할 수 있는 작업

정기배송 상세에서:

- **다음 배송일 변경** — 배송 예정일을 조정.
- **배송 주기 변경** — 구독 주기를 변경.
- **상태 변경** — 진행중 ↔ 일시정지 ↔ 해지.
- **변경 이력(감사 로그) 확인** — 누가 언제 무엇을 바꿨는지.
- **엑셀 내보내기** — 구독 목록을 `.xlsx`로.

> 정기배송 상태를 바꾸면 다음 결제·배송 일정에 영향을 준다. 고객 요청에 따라 신중히 처리한다.

## 4. 비즈 문의 응대

사이드바 **인사이트 > 비즈 문의**. (사업 제휴·대량 주문 등 문의)

### 상태 흐름

```
신규 → 연락중 → 종결
```

### 절차

1. 알림 벨의 "신규 비즈문의" 또는 **비즈 문의 > 신규** 탭에서 새 문의를 연다.
2. 문의 상세(회사명·연락처·내용)를 확인한다.
3. 고객에게 연락(메일·전화)한 뒤 상태를 **연락중**으로 변경.
4. 처리가 끝나면 상태를 **종결**로 변경.

> 상태 탭: 전체 / 신규 / 연락중 / 종결.

## 5. 뉴스레터 발송

사이드바 **인사이트 > 뉴스레터**.

### 5-1. 구독자 관리

구독자 목록에서 검색·필터·페이지 이동이 가능하다. 🔒 구독자 명단 CSV 내보내기는 **owner(관리자)만** 가능하다.

### 5-2. 뉴스레터 작성 → 발송 (실수 방지 절차)

1. 뉴스레터 **작성(컴포저)**에서 제목과 본문 블록(제목/문단/이미지/버튼)을 구성한다.
2. 오른쪽 **미리보기**로 실제 메일 모양을 확인한다.
3. **테스트 발송** — 본인 이메일로 먼저 보내 실제 수신 화면을 점검한다. **(발송 전 반드시 권장)**
4. 이상 없으면 🔒 **캠페인 발송** — 전체 활성 구독자에게 발송한다. **owner(관리자)만** 가능.

> ⚠️ 캠페인 발송은 **전체 구독자에게 한 번에** 나간다. 되돌릴 수 없으므로 반드시 테스트 발송으로 확인한 뒤 보낸다.
>
> 발송한 캠페인은 **발송 이력** 탭에서 확인하고, 필요 시 복제해 재발송할 수 있다.
> 각 메일에는 수신 거부(구독 해지) 링크가 자동 포함된다.

## 6. 상품 관리

사이드바 **카탈로그 > 상품**.

### 할 수 있는 작업

- **신규 상품 등록** — **상품 추가**(또는 새 상품) 버튼 → 이름·가격·설명·이미지·옵션 입력 후 저장.
- **상품 수정** — 목록에서 상품을 열어 정보·이미지 수정.
- **판매 노출 토글** — 활성/비활성 전환. 비활성 상품은 쇼핑몰에 안 보인다.
- **진열 순서 변경** — 목록에서 순서를 끌어 정렬.
- **상품 삭제** — 영구 삭제 (주의).

> 📷 **이미지 주의:** 상품 이미지는 업로드 후 자동 처리된다. 정적 이미지를 코드로 새로 추가하는 경우(개발 작업)는 별도 절차가 있으나, 운영자가 어드민에서 올리는 이미지는 그대로 처리된다.

## 7. 카페 메뉴 관리

사이드바 **카탈로그 > 카페 메뉴**.

상품과 동일한 방식으로:

- **신규 메뉴 등록** / **수정** / **삭제**
- **노출 토글** (활성/비활성)
- **이미지 업로드**
- **진열 순서 변경**

등록·수정한 메뉴는 사이트 **카페 메뉴** 페이지에 반영된다.

## 8. 사이트 설정

사이드바 **설정 > 사이트 설정**. 네 가지 영역을 한 화면에서 편집하고 **변경사항 저장**으로 반영한다.

### 8-1. 영업시간

- **위젯 표시 마스터 토글** — 켜면 사이트(스토리 페이지)에 영업시간이 표시되고, 끄면 숨겨진다.
- **휴무 요일** — 정기 휴무 요일을 칩으로 선택 (선택한 요일은 휴무로 처리).
- **영업일 시간** — 영업하는 요일별로 **시·분 드롭다운**(24시간제)으로 오픈/마감 시간 설정.
- **비정기 휴무** — 특정 날짜 + 사유를 추가 (명절·임시 휴무 등). **+추가** → 날짜·사유 입력.

### 8-2. 공지

사이트 상단 공지 바 문구·노출 여부를 설정.

### 8-3. 배송

배송 정책(배송비·무료배송 기준 등)을 설정.

### 8-4. 메인 노출 메뉴

홈 화면에 노출할 추천 메뉴를 드롭다운에서 선택한다.

> 저장 후에는 실제 사이트에서 반영을 확인한다.

## 9. 배너 / 굿데이즈 관리

### 9-1. 배너 — 사이드바 **설정 > 배너**

- **새 배너** 등록 / 수정 / 삭제.

### 9-2. 굿데이즈 — 사이드바 **카탈로그 > 굿데이즈** (이벤트·매장 사진 갤러리)

- **이미지 업로드** / 교체 / 삭제.
- 노출 **순서 변경**.

## 10. 회원 · 권한 관리 🔒 (owner 전용)

사이드바 **카탈로그 > 고객**.

> 이 장의 작업은 대부분 **owner(관리자)만** 가능하다. 신중히 다룬다.

- **회원 조회** — 검색·필터로 회원 확인.
- 🔒 **관리자 권한 부여 / 회수** — 일반 회원을 운영 계정으로 승격하거나 해제.
- 🔒 **권한 단계 변경** — owner ↔ staff. (마지막 owner는 자기 자신을 강등할 수 없다)
- 🔒 **회원 명단 내보내기** — CSV.
- 🔒 **계정 강제 삭제** — 회원 계정을 영구 삭제. **되돌릴 수 없으므로 극히 신중히.**

## 11. 통계 · 감사 로그 (조회 전용)

- **인사이트 > 통계** — 매출·주문 등 집계 조회.

## 12. 자주 묻는 운영 이슈 (FAQ)

**Q. 결제는 됐는데 발송 처리 버튼이 안 눌려요.**
→ 주문 상태가 **신규(결제완료)** 일 때만 발송 처리가 가능하다. 결제대기·이미 배송중인 주문은 비활성이다.

**Q. 환불 버튼이 어디에도 없어요.**
→ 정상이다. 환불은 어드민이 아니라 **Toss 콘솔**에서 한다 (2장 참고). 처리하면 주문 상태가 자동으로 바뀐다.

**Q. 송장번호를 잘못 입력했어요.**
→ 발송 처리는 한 번만 가능하다. 잘못된 송장 수정이 필요하면 개발 담당에게 문의한다. (알림 벨의 "송장 누락"은 송장이 비어 있는 경우다)

**Q. 캠페인 발송 버튼이 안 보여요.**
→ 뉴스레터 캠페인 발송은 **owner(관리자)** 권한이 필요하다. staff(운영자)에게는 보이지 않는다.

**Q. 영업시간을 바꿨는데 사이트에 안 보여요.**
→ **위젯 표시 마스터 토글**이 꺼져 있으면 사이트에 표시되지 않는다. 토글을 켜고 저장했는지 확인한다.

**Q. 알림 벨 숫자가 안 바뀌어요.**
→ 알림은 페이지 이동 또는 창으로 돌아올 때 갱신된다. 다른 메뉴를 한 번 눌렀다 돌아오면 최신화된다.
