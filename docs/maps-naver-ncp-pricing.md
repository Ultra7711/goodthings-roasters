# 네이버 지도 SDK(NCP Maps) 요금·약관 검토

> 작성: 2026-06-22 (S322) · 목적: "지도 SDK 카카오 → 네이버 전환" carry 의 **비용/가입 전제** 확정.
> ⚠️ 비용·약관 단정 금지 원칙 — 본 문서는 **공식 출처(ncloud.com / ncloud-forums.com / ncloud-docs)** 확인분과
> **미확인분**을 구분해 기재한다. 수치 인용 시 반드시 아래 출처 재확인 후 사용.

---

## 1. 질문과 답 (요약)

| 질문 | 답 |
|---|---|
| 네이버 지도 SDK 는 유료 전용인가? | **아니오.** 호출수 기준 **종량 요금제**이되 **무료 이용량**이 있다. |
| 가입이 필요한가? | **예.** `developers.naver.com`(로그인 API 키)이 아니라 **NAVER Cloud Platform(ncloud.com) 별도 가입** 필요. 현재 **미가입**. |
| 유료 가입자 한정인가, 대표계정 1개 무료인가? | **사업자등록번호(개인은 휴대폰번호)당 "대표계정" 1개에 한해** 각 API 월 무료 이용량 제공. **그 외 계정은 전부 과금.** |

---

## 2. 확인된 사실 (공식 출처)

- **요금 모델:** Maps API 는 **요청 호출수 기준 종량 요금제**.
  - Web Dynamic Map: **지도 로드 시점에만** 과금. 로드 후 zoom·마커 추가 등은 미과금.
  - Mobile Dynamic Map(SDK): API 호출이 아닌 **지도 뷰 생성 시점(onCreate)** 기준 카운트.
- **무료 이용량(대표계정):** API별 **월 최소 3,000건 ~ 최대 1억건** 무료 제공.
  - 무료 대상 명시 API: **Web Dynamic Map · Static Map · Geocoding · Reverse Geocoding** (4종).
  - "Maps API 를 가장 먼저 이용한 계정"이 **대표계정**으로 설정됨.
  - 식별 단위: **사업자등록번호**(사업자) / **휴대폰번호**(개인) 당 **대표계정 1개**.
- **무료 한도 합산 방식:** 앱(애플리케이션)별이 아니라 **계정 단위로 합산**. 여러 앱을 써도 무료 총량은 늘지 않고 풀(pool)에서 차감.
- **지출 상한:** NCP 콘솔에서 사용량 상한(spending cap) 설정 → **무료 범위 내로 제한 가능**.
- **대표계정 외 계정:** 자동 무료 한도 없음 → **전 사용량 과금**.
- **서비스 버전:** 기존 Maps API 가 **새 "Maps" 서비스로 업그레이드**됨(신규 이용 시 새 서비스 권장). VPC/Classic 지원, 한국 포함 5개 리전.
- 무료 정책 적용: **2023년 1월~**.

## 3. 미확인 — 가입/도입 전 반드시 공식 확인

> WebFetch 로 추출 실패(요금표가 동적 렌더). **아래 수치는 본 문서에 적지 않음. 추측 금지.**

- [ ] **API별 정확한 무료 건수**(3,000~1억 사이 어느 값인지 서비스별)
- [ ] **무료 초과분 호출당 단가(원)**
- [ ] **결제수단(신용카드) 등록이 가입/활성화에 필수인지** — NCP 종량 서비스는 통상 등록을 요구하나 공식 확인 필요
- [ ] Directions(길찾기) API 의 무료 대상 포함 여부 (위 4종에 명시 없음)

---

## 4. GTR 적용 판단

- 굳띵즈는 **단일 매장(1개 지점)** 위치 표시 용도 → 지도 로드는 **월 수백 회 수준** 추정.
- 가장 작은 무료 한도(월 3,000건)도 **실사용을 크게 상회** → **대표(사업자) 계정으로 무료 범위 내 운영이 현실적 결론**.
- 단, 가입 시 **결제수단 등록 요구 가능성** + 만일의 트래픽 급증 대비 **콘솔 지출 상한 설정**으로 과금 리스크 차단 권장.

## 5. 현재 구현 상태 (참고 · S320)

- 현재 지도 **SDK 는 카카오 유지**. 네이버는 말풍선 "상세보기/길찾기" **외부 링크**만 사용
  (`map.naver.com/p/entry/place/{naverPlaceId}` · `/directions`).
- "네이버 SDK 전환"은 **선택 과제** — 카카오 SDK 도 자체 무료 한도가 있어, 전환은 비용보다
  **온오프 시너지(네이버 검색→플레이스→통합적립)** 관점의 결정 사항.
- 전환 플랜 상세: `~/.claude/plans/warm-whistling-mountain.md` (레포 외부).
- 관련 결제 검토: `docs/payment-naverpay-easyconnect-review.md`.

## 6. 결론

네이버 지도 SDK 도입의 **비용 장벽은 사실상 없음**(대표 사업자 계정 무료 한도 >> GTR 사용량).
**진짜 선결 조건은 "NCP 가입 + (아마도) 결제수단 등록"** 이며, 가입 후 콘솔에서 발급되는
`ncpKeyId` 를 env 에 넣어야 한다. 도입 여부 자체는 비용이 아닌 **카카오 vs 네이버 전략 선택**의 문제.
